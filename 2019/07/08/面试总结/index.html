    • mysql 和 oracle分页语句
      mysql使用limit，oracle使用子查询
      mysql: select * from table limit (当前页码 - 1) * 每页记录数, 每页记录数;
      oracle: select * from (select *,ROWNUM RN from table where rownum <= 当前页码 * 每页记录数) where RN > ((当前页码数 - 1) * 每页记录数 + 1);

    • 常见设计模式有哪些，观察者模式及样例
      常见设计模式：原型、工厂、策略、代理、适配器、模板、迭代器、单例、观察者
      观察者模式：使用频率最高的设计模式，用于建立一种对象与种对象之间的依赖关系，一个对象发生改变时将自动通知其他对象，其他对象将作出反应。
      样例：java的图形编程中事件的监听就属于观察者模式。通过创建监听类并对组件注册监听，当组件发生相应事件，就会调用监听类中的方法。

    • map和set的区别
      map和set都是java中的对象容器， 其中，map以键值对形式存储，是一组映射关系,而set只有值，且set中的元素不可重复。

    • 线程池的理解
      线程池就是一个或多个线程的集合。如果并发的线程很多，且每个线程执行的时间很短，那么频繁的创建线程销毁线程就会大大降低系统效率。java中就是通过线程池实现线程的复用，就是一个线程执行完一个程序不被销毁，可以继续执行其他任务。
      创建线程池：举一个例子 Executors.newCachedThreadPool()创建ExecutorSevice对象，然后通过execute提交实现Runnable接口的任务。

    • java有哪些平台
      javaSE，javaEE，javaME

    • String，StringBuffer，StringBuilder
      String是java中的基础且最重要的类，它是不可变类。
      有些时候需要频繁拼接字符串，这样每次连接字符串都会构建一个新的String对象，既耗时又浪费空间。
      StringBuffer和StringBuilder可以解决这个问题，不同的是StringBuffer是线程安全的，而StringBuilder是线程不安全的，但是一般情况操作字符串不会涉及线程安全问题，所以更多使用StringBuilder。

    • map和set的区别
      map和set都是java中的对象容器， 其中，map以键值对形式存储，是一组映射关系,而set只有值，且set中的元素不可重复。

    • HashTable和HashMap的区别
      HashTable是线程安全的，HashMap是线程不安全的。
      HashTable不允许null值，HashMap允许null。
      HashTable直接使用对象的hashcode，而HashMap会重新计算hash值。

    • 比较两个数组是否相同
      使用Arrays的equals()方法,两个数组大小相等且下标相同的元素都对应相等。

    • java内存会不会泄露
      java理论上不存在内存泄露，因为有GC也就是垃圾回收机制会释放没有引用的对象
      但实际上也会有内存泄露的问题，就是由于一些对象虽然不再使用，但是仍然在被引用
